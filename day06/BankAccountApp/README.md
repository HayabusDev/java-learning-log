# Day06 - Bank Account System

## 作成物
コンソール銀行口座管理アプリ

## 学習内容
- クラス設計（Main / Account の責任分離）
- インスタンスの扱い方
- メソッドによる処理委譲
- 引数を使ったデータ受け渡し
- booleanによる成否管理
- 入金・出金ロジックの設計

## 気づき
- 入力値は一時データとしてMainで管理し、Accountには保存しない
- 残高の更新はAccountクラスに集約するべき
- 戻り値にエラーと金額を混ぜる設計はバグの原因になる
- 副作用のあるメソッドは1回だけ呼ぶ必要がある
- 成功/失敗と状態管理を分離すると設計が安定する

## 将来性・アップデート
- Listを使った複数口座管理
- ログイン機能の追加
- データベース（DB）との連携
- Spring Bootを用いたWebアプリ化
