# Day07 - Login System

## 作成物
コンソールログイン管理アプリ  
（複数ユーザー対応／新規登録機能付き）

## 学習内容
- HashMapによるユーザー情報管理
- MVC風クラス設計（Controller / Service / Repository / Model）
- コンストラクタによる依存関係の注入
- メソッド分離による責務管理
- 入力値による分岐処理

## 気づき
- データ管理と処理ロジックを分けることで、コードが整理しやすくなる
- Service層で判定処理をまとめると、Controllerが簡潔になる
- HashMapを使うことで、高速にユーザー検索ができる

## 改善点
- パスワードを平文で保存しているため、セキュリティ面に課題がある
- 入力エラー処理（数字以外入力など）が不十分
- ユーザー削除・変更機能が未実装
- データがメモリ上のみで、終了すると消える

## 今後の展望
- ファイル保存やDB連携による永続化対応
- パスワードの暗号化処理の導入
- GUI化またはWebアプリ化
- ログイン履歴や権限管理の追加
- Spring Bootを使ったAPI化への発展
